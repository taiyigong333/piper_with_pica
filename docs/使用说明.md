# 使用说明

本文是项目的长期操作入口。它不使用日期命名；每次新增或修改用户可见功能、配置字段、命令或现场流程后，必须同步更新本文，并在 `docs/` 中补充对应的日期文档。`项目交接.md` 只记录项目状态，不替代本说明。

## 环境

项目仅使用 uv 管理的 Python 3.10 虚拟环境，不修改系统全局 Python 或 `pika` Conda 环境：

```bash
# 在项目根目录创建/同步本项目的隔离环境。
UV_CACHE_DIR=/tmp/uv-cache uv sync --extra dev --extra realsense
```

## 常用命令

| 命令 | 作用 | 是否控制硬件 |
| --- | --- | --- |
| `uv run piper-collect discover-realsense --require-device` | 列出已连接的 RealSense 序列号。 | 否 |
| `uv run piper-collect preflight --config <采集 YAML>` | 读取相机、Piper、TCP、夹爪反馈，检查采集前提。 | 否 |
| `uv run piper-collect read-piper-state --config <采集 YAML>` | 读取当前六关节 rad 和原生 TCP，便于填写起始位姿。 | 否 |
| `uv run piper-collect collect --config <采集 YAML>` | 按 YAML 采集一条 HDF5 轨迹并生成质量文件。 | 是：采集读取；不发送运动 |
| `uv run piper-collect validate <trajectory.hdf5> --config <采集 YAML>` | 只读检查 HDF5 结构、同步、图像和质量。 | 否 |
| `uv run piper-collect trajectory-viewer --config <采集 YAML>` | 在浏览器同页同步查看所有相机帧、关节、TCP、夹爪和 `quality.json`。 | 否 |
| `uv run piper-collect calibrate-base --teleop-config <遥操 YAML> --mode force` | 执行 Pika 基站强制校准。 | Pika 定位设备 |
| `uv run piper-collect teleop-status` | 只读列出当前用户残留的 Pika 遥操进程。 | 否 |
| `uv run piper-collect teleop-stop --terminate` | 显式逐级关闭可识别的残留 Pika 遥操进程。 | 是：终止进程 |
| `uv run piper-collect teleop-session --config <采集 YAML> --teleop-config <遥操 YAML> --repeat` | 编排起始位姿、Pika 遥操、轨迹保留/删除与连续多条采集。 | 是 |

所有命令可加 `UV_CACHE_DIR=/tmp/uv-cache`，避免 uv 缓存目录权限或代理环境问题。例如：

```bash
UV_CACHE_DIR=/tmp/uv-cache uv run piper-collect trajectory-viewer \
  --config configs/pick_up_the_corn_and_put_it_on_the_plate.yaml
```

## 轨迹检查

每条轨迹正常完成后依次执行：

```bash
# 先进行自动只读校验。
UV_CACHE_DIR=/tmp/uv-cache uv run piper-collect validate <trajectory.hdf5> \
  --config <采集 YAML>
# 再启动浏览器查看器，检查图像和机器人状态是否同步。
UV_CACHE_DIR=/tmp/uv-cache uv run piper-collect trajectory-viewer \
  --config <采集 YAML>
```

详细界面说明见 [2026-07-20_02_轨迹可视化查看器.md](2026-07-20_02_轨迹可视化查看器.md)，遥操收尾与自动检查见 [2026-07-20_01_遥操收尾与轨迹检查.md](2026-07-20_01_遥操收尾与轨迹检查.md)。

## 遥操收尾

一条遥操轨迹必须按下面的顺序结束：第一次空格结束采集并关闭 HDF5；看到写盘完成提示后双击 Sense 停止实体遥操；最后按第二次空格，由编排层关闭 ROS 进程。开始下一条之前先运行 `teleop-status`；存在残留进程时，确认无关任务不受影响后执行 `teleop-stop --terminate`。

`--repeat` 模式仅在第一条开始前显示“已按现场情况完成基站校准”的确认；后续条目仍会完整执行预检、起始位姿与遥操重启，但不会重复询问校准。第一次空格会立即禁止新相机帧进入采集队列，终端随后显示 HDF5 收尾进度。第二次空格会并行关闭传感器与控制器的 ROS 进程组，减少两个收尾超时串行累加的等待。

每一条重复会话的 Sense RViz 与 Piper RViz 都必须重新弹出。若日志显示任一 `rviz2` 退出，编排层会拒绝显示“遥操节点已就绪”；当前已处理首条采集后 OpenCV 把 `.venv/cv2/qt/plugins` 传给下一条 ROS/RViz、导致 `xcb` 崩溃的问题。

每条轨迹完整写盘并关闭 ROS 后，按空格保留，按 `d` 删除，按 `q` 保留。删除会移除当前轨迹目录的 HDF5、质量报告、manifest 和校验和，适合排除不合格数据；它不会删除 `teleop_logs`。传入 `--on-complete save` 或 `--on-complete delete` 可跳过该交互。详细边界见 [2026-07-20_04_轨迹保留与删除.md](2026-07-20_04_轨迹保留与删除.md)。

ROS 收尾会在终端输出 `Pika ROS 关闭结果`，列出 `SIGINT`、`SIGTERM`、`SIGKILL` 各阶段的实际等待时间及剩余命令。控制器首轮只中断 `ros2 launch` 父进程，传感器脚本首轮中断进程组；详情、参数与试验验证方法见 [2026-07-20_05_ROS收尾优化试验.md](2026-07-20_05_ROS收尾优化试验.md)。
