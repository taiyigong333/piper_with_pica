# 轨迹保留与删除

`teleop-session` 完成一条轨迹后，先关闭 HDF5、写入 `quality.json`、`manifest.json` 和校验和，再关闭 ROS。此时终端显示：

```text
本条轨迹已写盘。按空格保留；按 d 删除；按 q 保留。
```

按空格保留目录中的完整数据包；按 `d` 删除这一条轨迹目录，用于排除画面、动作或同步不合格的数据；按 `q` 不删除，等同于保留。该选择不需要输入 `yes/no` 或 Enter。

删除动作仅允许作用于当前 YAML 的 `session.output_root` 内、当前会话生成的轨迹目录。目录外路径会被拒绝，防止错误删除其他数据。删除后本条 HDF5、质量报告、manifest 与校验和都会一并移除；对应的 `teleop_logs` 仍保留在输出根目录，便于追踪失败原因。

对于无人值守或明确的批量策略，可以跳过交互选择：

```bash
# 每条完成后直接保留，不等待处理选择。
UV_CACHE_DIR=/tmp/uv-cache uv run piper-collect teleop-session \
  --config configs/pick_up_the_corn_and_put_it_on_the_plate.yaml \
  --teleop-config configs/pika_sense_piper.example.yaml \
  --on-complete save

# 每条完成后直接删除，仅用于明确不需要保留的试运行。
UV_CACHE_DIR=/tmp/uv-cache uv run piper-collect teleop-session \
  --config configs/pick_up_the_corn_and_put_it_on_the_plate.yaml \
  --teleop-config configs/pika_sense_piper.example.yaml \
  --on-complete delete
```

在 `--repeat` 模式中，每条轨迹先完成上述保留/删除选择，再出现“准备下一条”的空格确认。终端会明确显示本条是“已保留”还是“已删除”。
