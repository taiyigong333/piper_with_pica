# 轨迹可视化查看器

`trajectory-viewer` 是本项目提供的本地只读 Web 界面，用于人工检查一条或多条已经完成的轨迹。它不启动 Piper、RealSense、Pika ROS，也不会修改 `trajectory.hdf5`、`quality.json` 或任何采集文件。

## 启动

在项目根目录执行。`--config` 会读取该 YAML 的 `session.output_root`，因此适用于现场采集配置：

```bash
# 在浏览器中查看该现场配置输出目录下的所有已完成轨迹。
UV_CACHE_DIR=/tmp/uv-cache uv run piper-collect trajectory-viewer \
  --config configs/pick_up_the_corn_and_put_it_on_the_plate.yaml
```

终端会输出 `http://127.0.0.1:8765`。在本机浏览器打开这个地址；完成检查后回到该终端按 `Ctrl-C` 停止服务。

也可以不读取 YAML，直接指定一个轨迹数据根目录：

```bash
# 只读查看指定目录下递归找到的 trajectory.hdf5。
UV_CACHE_DIR=/tmp/uv-cache uv run piper-collect trajectory-viewer \
  --root /home/ubuntu/gcj/projects/data_collect/data/corm_in_plate
```

默认仅监听 `127.0.0.1`，其他机器不能访问。只有确实需要从同一受信任局域网的另一台机器查看时，才显式指定 `--host 0.0.0.0` 并设置一个未被占用的 `--port`；数据中可能含现场图像，不应将该端口暴露到不受信任网络。

## 界面内容

- 顶部轨迹导航：下拉框选择数据根目录内的已完成 `trajectory.hdf5`，最新修改的轨迹在前；“上一条轨迹”和“下一条轨迹”按钮按此顺序切换，首尾按钮会自动禁用；刷新按钮会重新扫描目录。
- 相机帧：当前轨迹的所有 RGB 相机同时显示在同一个页面。现场的 D435 前视与 D405 腕部视角会共享同一帧序号和时间轴，无需切换；窄屏时两路画面在同一页面纵向排列。拖动时间轴、点击前后帧按钮，或使用左右方向键逐帧检查。
- 关节/TCP/夹爪：显示与相机帧对齐的状态值。TCP 标签依据轨迹自身的 `metadata/pose_representation` 标注；Piper 的 `xyz_rxryrz` 以 m + rad 显示。
- 时序总览：关节和 TCP 的全程曲线，图例使用与曲线一致的颜色标识每个数值。超过 2,000 个样本时，曲线会等间隔抽样以保持浏览器响应；逐帧数值仍来自完整对齐时间轴。
- TCP 详细时序：详细区首先显示位置 `x/y/z`（m）与 Piper 原生姿态 `rx/ry/rz`（rad）两张图。两张图均有与曲线同色的图例，避免位置与角度量纲混在同一纵轴中。
- 关节角详细时序：TCP 下方先显示六关节的合并详细图，便于比较关节间联动；随后显示六张独立纵轴图，适合观察单个关节的小幅变化。标题和颜色明确标注 `J1` 至 `J6`，单位均为 rad。
- 质量报告：显示同目录 `quality.json` 的自动检查结果、错误和告警。未找到该文件时会明确提示。

查看器只读取 `camera_observations/color_images`，不显示深度图；深度、时间轴、字段完整性和图像解码仍由 `piper-collect validate` 负责。

## 人工检查顺序

1. 先执行 `validate`，确认结果为 `pass`；再启动查看器。
2. 切换每一台相机并拖动时间轴，确认画面没有黑帧、错误相机或明显的时间跳变。
3. 在动作开始、关键抓取/放置时刻和结束处查看关节、TCP 和夹爪数值，确认它们与画面中的运动一致。
4. 查看质量报告中的重复帧、外参或时间轴告警，并在需要时填写轨迹目录中的人工质量记录。

出现结构性错误、画面与状态明显不同步或质量报告为 `fail` 时，不要将轨迹用于训练；保留原始目录和遥操日志，用 `validate` 及采集日志定位原因。
